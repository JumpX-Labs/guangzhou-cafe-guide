# 广州咖啡馆探店手册 —— API 作业

## 作业目标

通过本作业，你将学会：

1. 使用真实的 HTTP API 对线上数据进行增、查操作。
2. 理解前端数据来自后端 API，而非写死在代码里。
3. 使用 AI 辅助完成 API 调用与数据处理。
4. 将一组主题数据转换成 JSON，并构建一个类似咖啡馆手册的 API 站点。

## 任务一：使用 API 新增一家咖啡馆（POST /api/cafes）

### 要求

调用 `POST http://localhost:3001/api/cafes`，新增一家咖啡馆。

**关键规则**：咖啡馆名称里必须带上你的姓名，例如：

- `花园咖啡（张三）`
- `东山口小馆（李四）`
- `西关旧时光（王五）`

### 必须填写的字段

| 字段 | 示例 | 说明 |
|------|------|------|
| `id` | `zhangsan-garden-2026` | 唯一标识，建议使用 `姓名拼音-店名-年份` |
| `name` | `花园咖啡（张三）` | **必须带你的姓名** |
| `slug` | `zhangsan-garden-cafe` | URL 友好的英文标识 |
| `district` | `dongshankou` | 从已有街区中任选一个 |
| `address` | 越秀区某某路 1 号 | 真实或虚构地址 |
| `hours` | 10:00 - 22:00 | 营业时间 |
| `tags` | ["社区", "安静", "手冲"] | 至少 2 个标签 |
| `rating` | 4.5 | 0-5 评分 |
| `priceRange` | ¥30-50 | 人均消费 |
| `signatureDrinks` | ["招牌拿铁", "冷萃"] | 至少 1 个招牌饮品 |
| `tips` | ["周末人多"] | 至少 1 条探店提示 |
| `vibe` | 适合一个人发呆 | 氛围描述 |
| `description` | 这是一家... | 简介 |
| `imagePrompt` | A cozy coffee shop... | 图片生成英文 prompt |
| `recommendation` | 来这里晒太阳 | 一句话推荐 |

### 示例 curl

```bash
curl -X POST http://localhost:3001/api/cafes \
  -H "Content-Type: application/json" \
  -d '{
    "id": "zhangsan-garden-2026",
    "name": "花园咖啡（张三）",
    "slug": "zhangsan-garden-cafe",
    "district": "dongshankou",
    "address": "越秀区某某路 1 号",
    "hours": "10:00 - 22:00",
    "tags": ["社区", "安静", "手冲"],
    "rating": 4.5,
    "priceRange": "¥30-50",
    "signatureDrinks": ["招牌拿铁"],
    "tips": ["周末人多，建议上午去"],
    "vibe": "适合一个人发呆或看书",
    "description": "藏在东山口老小区里的安静咖啡馆。",
    "imagePrompt": "A cozy coffee shop in Guangzhou old neighborhood, morning light, wooden furniture, quiet atmosphere",
    "recommendation": "来这里晒太阳、看书。"
  }'
```

### 验证

调用成功后，刷新前端页面 `http://localhost:5173/explore`，找到你新增的咖啡馆，并截图保存。

## 任务二：使用 API 获取所有咖啡馆，然后做点什么（GET /api/cafes）

### 要求

调用 `GET http://localhost:3001/api/cafes`，获取所有咖啡馆数据后，完成以下任一任务：

**A. 数据分析**

统计并回答：

- 一共有多少家咖啡馆？
- 哪个街区的咖啡馆最多？
- 评分最高的 3 家店是哪些？
- 人均消费最低的店是哪家？

**B. 数据可视化**

用任何你熟悉的工具（Excel、Numbers、Python、Chart.js、ECharts，甚至手绘），把数据做成一张图表，例如：

- 各街区咖啡馆数量柱状图
- 评分分布图
- 价格区间饼图

**C. 生成一份推荐清单**

根据获取的数据，用 AI 辅助写一份“广州咖啡探店推荐清单”，包含：

- 本周推荐（3 家）
- 人均最低推荐（3 家）
- 评分最高推荐（3 家）
- 一句话理由

### 提示

你可以直接让 AI 帮你：

> “帮我把这段 JSON 数据整理成按街区统计的表格，并找出评分最高的 3 家店。”

> “帮我把这些咖啡馆数据生成一个 ECharts 柱状图代码。”

## 任务三：大作业 —— 构建你自己的主题 API 站点

### 选题

从以下 5 个主题中选择一个你最感兴趣的：

1. **演员资料站** —— 演员、代表作、照片、经纪公司等
2. **咖啡馆探店手册** —— 可以继续扩展本作业主题
3. **读书笔记库** —— 书名、作者、评分、读后感、标签
4. **线上课程资源库** —— 课程名、讲师、平台、难度、标签
5. **AI 工具推荐库** —— 工具名、用途、价格、官网、标签

### 要求

1. **把主题数据整理成 JSON 格式**（至少 8 条记录）。
2. **构建一个类似咖啡馆手册的 API**，至少包含 **3 个端点**，例如：
   - `GET /api/items`：获取全部数据
   - `GET /api/items/:slug`：获取单条数据
   - `POST /api/items`：新增数据
   - 鼓励补充 `PUT /api/items/:id` 和 `DELETE /api/items/:id`
3. **必须有独立的 API 使用文档页**，说明每个端点的请求/响应格式。
4. **前端页面**至少包含：
   - 首页（展示主题、推荐条目）
   - 列表页（筛选、搜索）
   - 详情页
   - 管理页（增删改查）
5. **使用 AI 辅助完成**：可以让 AI 帮你生成数据、设计界面、编写代码、调试 API。
6. **本地能运行前后端**，并可以通过 API 请求完成数据操作。
7. **提交时附上本地 API 调用截图**（curl、Postman、浏览器控制台均可）。

### 提交物

- 项目 GitHub 仓库链接

- 一份 README，说明：
  - 项目主题

## 常见问题

**Q：新增后刷新页面看不到？**  
A：确认后端服务 `npm run server` 已启动，然后刷新浏览器页面。

**Q：slug 重复怎么办？**  
A：API 会返回 409 错误，换一个唯一的 slug 即可。

**Q：如何修改自己新增的咖啡馆？**  
A：使用 `PUT /api/cafes/:id`，把 `:id` 换成你新增时的 `id`。

**Q：任务三可以组队吗？**  
A：可以 2-3 人组队，但 README 中需说明每人负责的部分。
